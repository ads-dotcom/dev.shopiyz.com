Shopiyz Admin API overview

Magaza, urun, siparis, musteri ve stok verilerini Shopiyz paneli disinda guvenli sekilde yonetmek icin REST tabanli ilk API katmani.

Magaza bazli erisim Her musteri magazasi kendi token, scope ve limitleriyle ayrilir.
Yetki kontrollu Tokenlar yalnizca verilen kaynaklara ve islemlere erisebilir.
Olceklenebilir baslangic Pagination, filtreleme ve rate limitler ilk surumun parcasi olacak.

Client libraries

Ilk dokumanlarda resmi SDK yerine HTTP uzerinden acik ve okunabilir ornekler kullanilir. Bu sayede endpoint yapisi netlesmeden gelistiriciyi bir kutuphaneye kilitlemeyiz.

  • cURL ornekleri terminalden hizli deneme icin temel kabul edilir.
  • JavaScript ve Node.js ornekleri uygulama ve worker senaryolarini kapsar.
  • PHP ornekleri Turkiye e-ticaret ekosistemindeki klasik entegrasyonlar icin tutulur.
  • API davranisini dogrudan gormek ve token/header sorunlarini ayiklamak icin en yalın baslangic.
  • Dashboard, backoffice ve edge worker icinden standart fetch kullanimi.
  • Server-side entegrasyonlar, cron isleri ve partner uygulamalari icin tokeni guvenli ortam degiskeninde tutar.
  • ERP, muhasebe, ajans panelleri ve klasik hosting entegrasyonlari icin basit HTTP client ornekleri.

Authentication

Admin API istekleri magaza bazli uretilen access token ile calisir. Token, istekte X-Shopiyz-Access-Token header'i olarak gonderilir.

  • Her token tek bir magaza veya izin verilen magaza grubu ile sinirlanir.
  • Scope mantigi read_products, write_orders gibi kaynak/aksiyon ayrimi yapar.
  • Tokenlar frontend koduna gomulmez; server, worker veya guvenli entegrasyon katmaninda saklanir.

Token nereden alinir?

Shopiyz hesabindan magazayi acin, sonra admin panelinde Ayarlar > Admin API bolumune gidip Admin veya Storefront token olusturun. Token sadece olusturuldugu anda tam gosterilir.

Hesabi ac Development token ekrani 
curl -X GET "https://{store}.shopiyz.com/admin/api/shop.json" \
  -H "Content-Type: application/json" \
  -H "X-Shopiyz-Access-Token: {access_token}"

Rate limits

Admin API trafiği musteri magazasindan bagimsiz bir entegrasyon katmaninda sinirlandirilir. Boylece yogun API kullanimi storefront sayfa yukunu dogrudan artirmaz.

  • Limitler app + magaza bazinda hesaplanir.
  • GET agirlikli public/storefront istekleri cache ve CDN katmanina ayrilabilir.
  • Limit asildiginda 429 Too Many Requests ve Retry-After doner.
HTTP/1.1 429 Too Many Requests
X-Shopiyz-Api-Call-Limit: 60/60
Retry-After: 2

{
  "error": {
    "code": "rate_limited",
    "message": "Too many requests. Retry after 2 seconds."
  }
}

Status and error codes

Tum hata yanitlari ayni JSON semasini kullanir. Bu bolum API yayina hazirlandikca daha ayrintili hata kataloguna donusecek.

Kod Anlam Ne zaman doner?
200 OK Listeleme, detay ve basarili guncelleme yanitlari.
201 Created Yeni kaynak basariyla olusturuldugunda.
400 Bad Request Parametre veya JSON formati hatali oldugunda.
401 Unauthorized Token eksik, gecersiz veya suresi dolmus oldugunda.
403 Forbidden Token var ama gerekli scope veya magaza yetkisi yoksa.
422 Unprocessable Entity Kaynak dogrulama kurallarini gecemediginde.
429 Too Many Requests Rate limit doldugunda ve bekleme suresi gerektiginde.

Resource reference

Shopiyz Admin API kaynak haritasi genis platform mantiginda, ama ilk surum icin daha okunabilir ve REST odakli tutulur. Bu bolum canli endpointleri, planlanan pathleri ve dokumanlarda yer alacak temel alanlari gosterir.

Apps endpointleri ayri kartlarda Product Tab Manager, Product FAQ Studio, XML Product Import, Shopify Connection, Shopify Checkout, Faprika Connection, Digital Downloads ve Insert Code Studio ayni Apps basligi altinda, ama farkli uygulamalardir. Her uygulamanin pathleri kendi kartinda durur; diger kaynak aileleri kapsam genisledikce canli hale gelir.
